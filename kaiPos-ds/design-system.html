Design System · Material UI Theme

A calm, reliable interface
for six kinds of shift.

kaiPOS runs on counter-top terminals, steamy kitchens, waiters' phones, diners' own devices, managers' laptops, and an online storefront. One token system. One createTheme(). Touch-first by default, desktop-dense where it should be.

01

Six surfaces, one theme

Each app inherits the same token set, then tunes density via size variants — not a fork of the palette.

01 · POS Terminal

Counter-top checkout

15–22″ touch screen. Landscape. Gloves occasionally. Speed of keypad entry matters more than beauty.

size=pos 56px targets · radius-md · shadow-xs
02 · Kitchen Display

KDS — the hot line

Dark surface, high-contrast order cards, color-coded state. Designed for the furthest rack of the pass.

size=kds 64px · dark canvas · ticket variant
03 · Waiter mobile

On-the-floor ordering

Phone, single-handed. Sticky bottom actions. Scrollable menu with tile buttons.

size=large 48px min · radius-md · swipeable
04 · Customer PWA

Scan-to-order

Diners on their own phones. Lightweight, no login, venue-branded through palette overrides.

size=large warm theme · radius-lg · illustrated
05 · Admin dashboard

Manager web app

Desktop, pointer-first. Information-dense tables, dashboard charts, settings trees.

size=medium 36px · radius-sm · side-nav
06 · Online store

Public storefront

Marketing context. More negative space, bigger hero type, optional warm-amber accent on CTAs.

size=large 48px · radius-xl · editorial
